Key Facts

Linear search checks each item one by one from the start. Works on any list — sorted or unsorted. O(n) worst case.
Binary search halves the search space each step by comparing the middle item. Requires a sorted list. O(log n) worst case.
Exam tip: Binary search is much faster for large lists but only works if the data is already sorted. You may be asked to explain why.
