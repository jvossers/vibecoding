See how compilers and interpreters translate source code differently.

Key Facts

High-level languages (Python, Java) are human-readable and portable across different CPUs. They must be translated to machine code before execution.
Low-level languages include assembly (mnemonics) and machine code (binary). They are CPU-specific and harder to write but very fast.
A compiler translates the entire program at once before execution. It produces an object code file that can be run repeatedly without retranslation.
An interpreter translates and executes one line at a time. No object code is saved — the source must be re-interpreted every time it runs.
If there's an error: a compiler reports it before any code runs. An interpreter stops at the error line — previous lines may have already executed.
Exam tip: Compiler = faster runtime (object code), slower development (compile after every change). Interpreter = slower runtime (retranslates each time), faster development (immediate feedback, easier to debug).
Based on the OCR J277 specification.
