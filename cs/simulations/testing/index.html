Key Facts

A syntax error is a mistake in the grammar/rules of the language. The program will NOT run. E.g., missing brackets, colons, or misspelled keywords.
A logic error means the program runs but produces incorrect results. E.g., wrong formula, incorrect comparison, off-by-one error.
Normal test data — typical, valid values that the program should handle correctly.
Boundary test data — values at the edges of valid ranges (e.g., 0, 1, 99, 100 for range 0-100).
Invalid test data — values outside the valid range but of the correct type (e.g., -1, 101).
Erroneous test data — completely wrong data type (e.g., text when expecting a number).
Exam tip: For boundary testing of range 1-100, always test: 0 (just below), 1 (lower boundary), 2 (just above lower), 99 (just below upper), 100 (upper boundary), 101 (just above). This shows the examiner you understand boundaries.
Based on the OCR J277 specification.
