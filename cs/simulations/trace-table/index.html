Key Facts

          

            Trace table — A trace table records the values of variables as a program executes, line by line. It is used to dry-run code and find errors.
          

          

            Columns & rows — Each column represents a variable (or output). Each row represents one step of execution.
          

          

            Logic errors — Trace tables help you identify logic errors — when the program runs but produces wrong results.
          

          

            Exam tip: In trace table questions, work through EVERY line — don't skip steps. Show all variable changes, even when a variable doesn't change (repeat the value or leave blank). Always include an Output column for PRINT statements.
